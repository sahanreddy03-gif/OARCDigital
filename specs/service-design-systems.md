- Horizontal scroll carousel (smooth requestAnimationFrame)

- Fade-in on scroll for sections

## Global Design Principles (All Pages)

### Visual-First Approach
- **60% less text** than Phase 1 implementation
- **Headlines:** 6-12 words maximum
- **Body copy:** 1-2 sentences per section
- **Let images tell the story**

### Responsive Breakpoints
- **Mobile:** 375px - 767px (stack all sections vertically)
- **Tablet:** 768px - 1279px (2-column layouts)
- **Desktop:** 1280px+ (full asymmetric grids, multi-column)

### Performance
- **Images:** Lazy loading, WebP/AVIF formats
- **Animations:** CSS transforms, 60fps minimum
- **Videos:** Autoplay muted, poster images as fallbacks

### Accessibility
- **Contrast ratios:** WCAG AA minimum (4.5:1 for text)
- **Focus states:** Visible keyboard navigation
- **Alt text:** Descriptive for all images
- **Aria labels:** Proper semantic HTML

## Implementation Notes

1. **No Reusable Components:** Each service page gets its own dedicated page component
2. **Unique Layouts:** Do not force all pages into Hero → Grid → FAQ structure
3. **Asset Placeholders:** All images will be placeholder divs with specs until real assets are sourced
4. **Color Variables - CRITICAL:** 
   - Service-specific colors shown above use hex notation for reference only
   - When adding CSS variables to `index.css`, MUST use H S% L% format (space-separated with percentages)
   - Example: `--social-primary: 262 83% 58%;` (not `#8B5CF6`)
   - Reference existing index.css for proper formatting
5. **Mobile-First:** Build mobile view first, then enhance for desktop
6. **Image Resolution Standards:**
   - 4K/UHD = 3840x2160 (hero images, main visuals)
   - QHD+ = 2560x1440 (portfolio screenshots)
   - Full HD+ = 1920x1080 (standard content images)
   - All images should be served as WebP/AVIF with lazy loading
